OpenAPI import + Markdown notes + team sharing

Turn API definitions into maintainable team documentation

Simple API Doc provides a self-hosted API documentation workspace. It imports interfaces from Swagger/OpenAPI specs and combines Markdown notes, schema trees, online debugging, sharing permissions, and scheduled synchronization to help teams maintain readable long-term API assets.

Features

Core capabilities for API documentation maintenance

It is not just a Swagger UI replacement. It builds a complete loop around import, explanation, debugging, sharing, and version maintenance.

OAS

Swagger/OpenAPI Import

Import Swagger2/OpenAPI3 documents from JSON, YAML files, or URLs while preserving APIs, models, auth, and folder structure.

TREE

Nested Folders and Model Trees

Manage APIs, folders, data models, and components together. Suitable for layered reading and maintenance of large schemas.

MD

Markdown Supplementary Docs

Add pages beyond API fields, such as onboarding flows, authentication rules, error codes, business constraints, and Mermaid diagrams.

DBG

Online API Debugging

Configure request params, headers, body, authentication, and environment URLs. Choose browser sending or server proxy sending by scenario.

SHR

Secure Sharing

Share links can define password, expiration, copyright, watermark, export permission, and debug permission, exposing only the intended docs.

SYNC

Scheduled Import Tasks

URL imports can be saved as sync tasks and refreshed later by fixed frequency or manual trigger, reducing repetitive maintenance.

HIS

History and Locking

API docs and models support history viewing, diff comparison, and recovery. Important docs can be locked to avoid task overwrites.

GEN

Client Code Generation

Integrates OpenAPI Generator, supports selected API code generation, and can use proxy mode for restricted network environments.

ACL

Users and Project Groups

Supports user management, project groups, and read/write/delete permissions. Admins can manage tasks, logs, and DB pools centrally.

Workflow

Recommended workflow from API definitions to team docs

Import machine-readable OpenAPI first, add human-readable explanations next, then publish and continuously sync through sharing and tasks.

Import a project

Choose file import or URL import in API Projects, configure authentication, duplicate strategy, and folder rules, then generate projects, APIs, and models.

Organize documentation

Use folders, Markdown docs, environment configs, authentication configs, and model components to add business context. Lock manual content when necessary.

Debug and verify

Configure request parameters in API details and send requests with environment URLs, JWT, Basic, Token, and other authentication methods.

Publish shares

Create share links and restrict document scope and permissions. Expose only the content that should be viewed, debugged, or exported.

Keep synchronized

Create scheduled import tasks for URL sources and combine history with locking to handle conflicts between automatic updates and manual notes.

Quick Start

Start a private API documentation service with one command

H2 file storage is used by default. For long-term team deployment, mount the data directory or switch to MySQL.

Docker Startup

docker run -d --name simple-api-doc \
  -p 9089:9089 \
  -v apidoc-data:/data \
  fugary/simple-api-doc:latest

Login

Open http://localhost:9089/. The default account is admin/12345678.

Change the password after first login and choose H2 or MySQL based on team size.
