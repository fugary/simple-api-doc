OpenAPI 导入 + Markdown 补充 + 团队分享

把接口定义沉淀成可维护的团队文档

Simple API Doc 提供一个可私有化部署的 API 文档工作台。它可以从 Swagger/OpenAPI 文档导入接口，结合 Markdown 说明、模型树、在线调试、分享权限和自动同步，帮助团队维护长期可读的接口资产。

私有部署

支持发布包、Docker、H2 单机和 MySQL 团队部署。

Features

面向接口文档维护的核心能力

不是只展示 Swagger UI，而是围绕导入、补充说明、调试、分享和版本维护建立完整闭环。

OAS

Swagger/OpenAPI 导入

支持从 JSON、YAML 文件或 URL 导入 Swagger2/OpenAPI3 文档，并保留接口、模型、认证和目录结构。

TREE

多级目录和模型树

接口、文件夹、数据模型和组件集中管理，适合大型 schema 的分层阅读和团队维护。

MD

Markdown 补充文档

在接口文档之外新增说明页，补充接入流程、鉴权规则、错误码、业务约束和 Mermaid 图表。

DBG

在线调试接口

支持请求参数、请求头、请求体、认证信息和环境地址配置，浏览器发送或服务端代理发送可按场景选择。

SHR

安全分享

分享链接可设置密码、有效期、版权、水印、可导出和可调试选项，只开放需要发布的文档范围。

SYNC

自动导入任务

从 URL 导入时可保存同步任务，后续按固定频率或手动触发刷新文档，减少重复维护成本。

HIS

历史版本和锁定

接口文档和模型支持历史版本查看、差异对比和恢复，关键文档可锁定，避免被导入任务覆盖。

GEN

客户端代码生成

集成 OpenAPI Generator，支持按接口选择生成客户端代码，并通过代理方式适配受限网络环境。

ACL

多用户和项目分组

支持用户管理、项目分组和读写删权限控制，管理员可统一管理任务、日志和数据库连接池。

Workflow

从接口定义到团队文档的推荐流程

先导入机器可读的 OpenAPI，再补齐人可读的说明，最后通过分享和自动同步持续交付。

导入项目

在 API 项目中选择文件导入或 URL 导入，配置认证、重复策略和目录规则，生成项目、接口和模型。

整理文档

通过文件夹、Markdown 文档、环境配置、认证配置和模型组件补充业务上下文，必要时锁定手工维护内容。

调试验证

在接口详情中配置请求参数并发送请求，通过环境地址、JWT、Basic、Token 等认证方式验证接口可用性。

发布分享

创建分享链接，限制文档范围和权限；对外只暴露需要查看、调试或导出的内容。

持续同步

为 URL 来源创建自动导入任务，结合历史版本和锁定机制处理自动更新与人工说明之间的冲突。

Quick Start

一条命令启动私有 API 文档服务

默认使用 H2 文件数据库。团队长期部署时建议挂载数据目录或切换 MySQL。

Docker 启动

docker run -d --name simple-api-doc \
  -p 9089:9089 \
  -v apidoc-data:/data \
  fugary/simple-api-doc:latest

登录信息

访问 http://localhost:9089/，默认账号为 admin/12345678。

首次登录后请修改密码，并根据团队规模选择 H2 或 MySQL。
