Simple API Doc 使用指南

本文档按实际使用路径组织：先部署启动，再导入 OpenAPI 文档，随后维护 Markdown、模型、权限、分享和自动同步。系统默认端口为 9089，默认账号为 admin/12345678。

Docker 快速启动

默认使用 H2 文件数据库。为了避免容器重建后数据丢失，建议挂载 /data。

docker run -d --name simple-api-doc \
  -p 9089:9089 \
  -v apidoc-data:/data \
  fugary/simple-api-doc:latest

启动后访问 http://localhost:9089/。

默认账号密码为 admin/12345678。生产环境首次登录后应立即修改密码。

发布包运行

从 GitHub Releases 下载最新压缩包并解压，进入 bin 目录运行启动脚本。

Windows

bin\start.bat

Linux

chmod +x bin/start.sh
bin/start.sh

发布包内通常包含可执行 Jar、config/application.yml、bin 启动脚本和运行日志目录。

MySQL 配置

团队长期使用时建议切换 MySQL。Docker 部署可通过环境变量覆盖默认数据库配置。

docker run -d --name simple-api-doc \
  -p 9089:9089 \
  -e DB_TYPE=mysql \
  -e DB_MYSQL_SERVER=host.docker.internal \
  -e DB_MYSQL_PORT=3306 \
  -e DB_MYSQL_DBNAME=apidoc-db \
  -e DB_USERNAME=root \
  -e DB_PASSWORD=12345678 \
  fugary/simple-api-doc:latest
变量 说明
DB_TYPE 数据库类型，支持 h2 和 mysql。
DB_DATA_DIR H2 文件数据库目录，Docker 默认建议映射到 /data。
DB_MYSQL_SERVER MySQL 主机地址。
DB_MYSQL_PORT MySQL 端口，默认 3306。
DB_MYSQL_DBNAME 数据库名称，默认 apidoc-db。
DB_USERNAME / DB_PASSWORD 数据库账号和密码。
API_TASK_ENABLED 是否开启后台定时任务（包括日志及缓存清理），默认 true。
API_LOG_RETENTION_DAYS 访问日志保留天数，默认 180，<=0 时不自动清理。
API_AI_ENABLED 是否启用 AI 生成数据能力，默认 true。
API_AI_BASE_URL AI 模型 Base URL。
API_AI_API_KEY AI 模型 API Key。
API_AI_MODEL AI 模型名称。
API_AI_CACHE_RETENTION_DAYS AI 样本缓存过期清理天数，默认 90。

本地源码构建

项目由 Vue3 前端和 Spring Boot 后端组成。根目录脚本会先构建 UI，再复制到后端静态资源目录，最后执行 Maven 打包。

# Linux/macOS
./build.sh

# Windows
build.bat

也可以分步执行：

cd simple-api-doc-ui
npm install
npm run build
cd ..
mvn clean install -Dmaven.test.skip=true -Pproduction

登录和首页

访问服务地址后进入登录页，默认账号为 admin/12345678。登录后左侧菜单包含 API 管理、系统管理和常用工具。

  • API 管理：API 项目、分享管理、导入管理、项目分组。
  • 系统管理：用户管理、任务管理、日志管理、数据库连接池、H2 数据库。
  • 常用工具：图标管理、代码编辑器、差异对比编辑器。

创建或导入项目

进入 API 项目 后，可以新建空项目，也可以从 Swagger/OpenAPI 导入。

选择导入方式

支持文件导入和 URL 导入，文件通常为 .json 或 .yaml。

配置认证和来源

URL 导入可配置 Basic、Token、JWT 等认证信息，用于拉取受保护的 OpenAPI 文档。

处理重复接口

导入已有项目时，根据需要选择覆盖、跳过或中止重复 API。

保存自动任务

URL 导入成功后可保存为自动导入任务，后续按配置频率同步。

项目结构

一个 API 项目通常包含基础信息、文件夹、接口文档、Markdown 文档、数据模型、环境地址和分享配置。

对象 用途
项目 一组接口文档的顶层容器，包含项目编码、名称、图标和环境配置。
文件夹 用于组织接口和 Markdown 文档，支持多级树。
API 文档 接口路径、方法、请求参数、请求体、响应体、认证要求和示例。
Markdown 文档 补充业务说明、接入指南、错误码、流程图等非接口字段内容。
数据模型 OpenAPI 组件模型，支持 schema 字段查看和示例生成。

API 文档查看

接口详情页按请求和响应组织字段，支持路径参数、Query 参数、Header、Body、状态码、认证信息、示例和 schema 树查看。

对于大型 schema，建议使用树形或表格视图逐层展开，减少一次性渲染过多节点造成的阅读压力。

Markdown 文档

Markdown 文档适合维护接口之外的上下文信息，例如鉴权说明、业务流程、错误码表、版本变更和接入 FAQ。

  • 可与 API 文档放在同一目录树中统一分享。
  • 支持 Markdown 编辑和预览，适合补齐 OpenAPI 无法表达的业务语义。
  • 导入文档时可结合扩展字段保留附加 Markdown 文件。

数据模型和组件

项目组件用于维护 OpenAPI schema、认证配置和安全要求。系统支持查看模型属性、枚举、默认值、格式、正则、长度限制、数组子项和引用关系。

当模型被多个接口复用时，建议优先维护组件模型，再在接口请求体或响应体中引用，避免重复定义。

历史版本和锁定

API 文档和数据模型支持历史版本列表、差异对比和恢复。对于手工调整过的重要内容，可以使用锁定功能，避免后续自动导入覆盖。

推荐规则：自动生成字段交给导入任务维护，人工补充的 Markdown 和关键说明使用锁定或单独文档维护。

项目分组权限

项目分组可以为用户配置不同访问能力，包括禁止访问、可读、可写、可删除和完全访问。管理员可以统一管理分组成员。

多人协作时，建议按业务线或系统域拆分项目分组，再把项目归属到对应分组中。

分享文档

分享管理用于生成对外文档链接。分享可以限制访问范围和交互能力。

  • 可设置访问密码和过期时间。
  • 可选择分享全部文档或指定文档。
  • 可控制是否允许导出、是否允许调试。
  • 可配置版权信息、水印、树图标和子节点展示数量。

分享地址格式通常为 /share/{shareId}。

在线调试

接口详情和分享页可启用请求调试。调试支持环境地址、认证方式、请求头、Query、Path、Body 和响应预览。

发送方式 适用场景
浏览器发送 目标接口支持 CORS 时使用，响应直接由浏览器发起。
服务端发送 目标接口不支持跨域或需要通过服务端代理访问时使用。

导出和代码生成

项目和分享文档支持导出，适合离线交付或沉淀归档。系统也集成 OpenAPI Generator，可按语言或框架生成客户端/服务端代码。

代码生成依赖 OpenAPI 结构质量。生成前建议先检查 schema 引用、认证配置和接口选择范围。

自动导入任务

URL 导入可以保存为项目任务，后续自动拉取源 OpenAPI 文档并更新项目。任务支持手动触发、停止、禁用和查看执行状态。

管理员可在系统任务管理中查看运行中的任务实例、执行状态和异常信息。

数据和日志

  • H2 默认数据目录可通过 DB_DATA_DIR 配置，Docker 推荐挂载到 /data。
  • 日志默认输出到 logs/simple-api-doc，也可通过 LOG_DIR 覆盖。
  • 管理员可以在日志管理中查看操作日志，在数据库连接池中查看连接状态。
  • H2 控制台可通过 DB_H2_CONSOLE=true 开启，仅建议在受控环境使用。

常见问题

导入的接口太多，页面阅读困难怎么办？

优先使用目录树拆分业务模块，并通过 Markdown 文档维护入口说明。大型 schema 可按模型树逐层查看。

自动导入会覆盖手工修改吗？

导入策略决定重复接口如何处理。对于不能覆盖的内容，建议使用 Markdown 独立维护或锁定关键文档。

分享页能否允许对方调试接口？

可以。在分享配置中开启调试权限，并配置环境地址。若目标接口存在跨域限制，使用服务端发送。

为什么 Docker 容器重建后数据不见了？

H2 文件数据库需要持久化 /data。未挂载数据卷时，数据会随容器删除而丢失。
